All API requests require an API key passed as a query parameter.

Including Your API Key

GET http://localhost:8001/api/v1/memories?api_key=YOUR_API_KEY

Python Example

import os
import requests

API_KEY = os.environ.get('SECOND_BRAIN_API_KEY')
BASE_URL = 'http://localhost:8001/api/v1'

def make_request(endpoint, method='GET', **kwargs):
    url = f"{BASE_URL}{endpoint}"
    params = kwargs.get('params', {})
    params['api_key'] = API_KEY
    kwargs['params'] = params
    
    response = requests.request(method, url, **kwargs)
    response.raise_for_status()
    return response.json()

# Example: Create a memory
memory = make_request('/memories', method='POST', json={
    'content': 'Python is a versatile programming language',
    'memory_type': 'semantic',
    'tags': ['programming', 'python']
})

JavaScript Example

const API_KEY = process.env.SECOND_BRAIN_API_KEY;
const BASE_URL = 'http://localhost:8001/api/v1';

async function makeRequest(endpoint, options = {}) {
    const url = new URL(`${BASE_URL}${endpoint}`);
    url.searchParams.append('api_key', API_KEY);
    
    const response = await fetch(url, {
        headers: {
            'Content-Type': 'application/json',
            ...options.headers
        },
        ...options
    });
    
    if (!response.ok) {
        throw new Error(`API error: ${response.statusText}`);
    }
    
    return response.json();
}

// Example: Search memories
const results = await makeRequest('/memories/search', {
    method: 'POST',
    body: JSON.stringify({
        query: 'machine learning',
        limit: 10
    })
});

Security Best Practices

  • Never commit API keys to version control
  • Use environment variables for key storage
  • Rotate keys every 90 days
  • Use separate keys for development and production
  • Implement IP whitelisting for production keys
